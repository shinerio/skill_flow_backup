# 异常场景处理

本文档定义 SDD 设计过程中各种异常场景的处理方式，确保 skill 在遇到问题时能够优雅地处理并引导用户解决。

---

## 场景 1：全量规格文档缺失

### 触发条件

- 步骤 0 检测时，`{root}/codespec/specs/spec.md` 不存在
- 步骤 2 执行时，无法读取全量规格文档

### 处理流程

```markdown
⚠️ 未找到全量规格文档

路径：./codespec/specs/spec.md

全量规格文档是增量设计的基础参考，缺失会影响设计质量。

建议选项：
- [ ] 使用 CodeWiki 生成专业文档并手工归档
  1. 访问 https://codewiki.rnd.huawei.com/projects
  2. 创建项目并生成规格文档
  3. 下载并归档到 ./codespec/specs/spec.md

- [ ] 基于 code-explorer 分析代码生成规格
  1. 使用 code-explorer skill 分析代码结构
  2. 基于代码生成规格文档
  3. 归档到 ./codespec/specs/spec.md

- [ ] 手动提供规格文档路径
  请提供全量规格文档的完整路径：

- [ ] 基于代码仓直接工作（不推荐）
  ⚠️ 注意：没有全量规格文档作为参考，设计质量可能受到影响

请选择处理方式：
```

### 用户选择后的处理

**选择 CodeWiki**：
```markdown
✅ 已选择使用 CodeWiki 生成文档

请按以下步骤操作：
1. 访问 CodeWiki 并生成文档
2. 完成后回复"已完成"

等待完成后，将自动继续步骤 0。
```

**选择 code-explorer**：
```markdown
✅ 已选择基于 code-explorer 生成文档

正在调用 code-explorer skill 分析代码...

[等待 code-explorer 完成]

代码分析完成，正在生成规格文档...

[生成文档]

✅ 规格文档已生成并归档到 ./codespec/specs/spec.md

继续步骤 0...
```

**选择手动提供路径**：
- 验证路径是否存在
- 如果存在，使用该路径继续
- 如果不存在，提示重新输入

**选择直接工作**：
```markdown
⚠️ 警告：将在没有全量规格文档的情况下继续

这可能导致：
- 无法验证增量规格与全量规格的兼容性
- 可能遗漏已有的业务规则
- 设计质量可能受到影响

确认继续吗？
- [ ] 确认继续
- [ ] 返回重新选择
```

## 场景 3：工作目录不存在

### 触发条件

- 步骤 0 检测时，工作目录 `{root}/codespec/changes/{work_dir}/` 不存在
- 用户跳转到某个步骤时，工作目录已被删除

### 处理流程

```markdown
⚠️ 工作目录不存在

路径：./codespec/changes/US202601010015-增加逻辑多租

工作目录可能已被删除或从未创建。

建议选项：
- [ ] 创建新目录
  目录名格式：US编号-功能名
  例如：US202601010016-新功能名称

- [ ] 选择现有目录
  以下目录可用：
  - [ ] US202601010015-增加逻辑多租
  - [ ] US202601010017-性能优化

- [ ] 取消操作
  返回主菜单

请选择处理方式：
```

### 处理逻辑

**选择创建新目录**：
- 提示输入目录名
- 验证目录名格式（US编号-功能名）
- 创建目录及子目录结构
- 继续步骤 0

**选择现有目录**：
- 加载所选目录的文档
- 识别当前进度
- 询问用户是继续还是重新开始

**选择取消**：
- 返回主菜单或等待用户指令

## 场景 4：用户回答冲突

### 触发条件

- 用户当前回答与之前的回答矛盾
- 例如：之前说"目标用户是内部员工"，现在说"目标用户是外部客户"

### 处理流程

```markdown
⚠️ 检测到回答冲突

您之前提到：
- "目标用户：内部员工"

现在又说：
- "目标用户：外部客户"

这两个回答存在矛盾，请问以哪个为准？

- [ ] 之前的回答（内部员工）
- [ ] 现在的回答（外部客户）
- [ ] 两者都是（请说明场景）
- [ ] 我重新回答
```

### 处理逻辑

**选择之前的回答**：
```markdown
✅ 已选择之前的回答

目标用户：内部员工

继续下一轮问答...
```

**选择现在的回答**：
```markdown
✅ 已选择现在的回答

目标用户：外部客户

⚠️ 注意：之前的理解已更新，可能影响之前的设计建议。

是否需要重新评估之前的建议？
- [ ] 是，重新评估
- [ ] 否，继续
```

**选择两者都是**：
```markdown
✅ 已选择两者都是

请说明不同场景：

- 场景 1：[场景描述] → 目标用户：内部员工
- 场景 2：[场景描述] → 目标用户：外部客户

继续下一轮问答...
```

**选择重新回答**：
- 清除该问题的记录
- 重新提问

## 场景 5：用户回答不完整

### 触发条件

- 用户只回答了部分问题
- 用户跳过了某些问题

### 处理流程

```markdown
📋 回答确认

我注意到您没有回答以下问题：

1. [未回答的问题1]
2. [未回答的问题2]

这些信息对设计很重要，请问：

- [ ] 现在补充回答
- [ ] 暂时跳过，后续补充
- [ ] 使用默认值：[默认值说明]

请选择：
```

### 处理逻辑

**选择现在补充**：
- 重新提问未回答的问题
- 等待用户回答

**选择暂时跳过**：
```markdown
✅ 已暂时跳过

这些问题将在后续阶段再次确认。

继续下一轮问答...
```

**选择使用默认值**：
```markdown
✅ 已使用默认值

问题1：[默认值]
问题2：[默认值]

⚠️ 注意：默认值可能不准确，建议后续确认。

继续下一轮问答...
```

## 场景 6：文档生成失败

### 触发条件

- 生成文档时发生错误
- 无法写入文件
- 模板文件缺失

### 处理流程

```markdown
❌ 文档生成失败

错误信息：[具体错误信息]

可能的原因：
- 文件权限问题
- 磁盘空间不足
- 模板文件缺失

建议选项：
- [ ] 重试生成
- [ ] 检查文件权限
- [ ] 手动创建文档（提供内容）
- [ ] 跳过此步骤

请选择处理方式：
```

### 处理逻辑

**选择重试**：
- 重新尝试生成文档
- 如果再次失败，提示其他选项

**选择检查文件权限**：
```markdown
📋 检查文件权限

请执行以下操作：
1. 检查目录权限：`ls -la ./codespec/changes/{work_dir}/`
2. 确保有写入权限

完成后回复"已检查"，将重试生成。
```

**选择手动创建**：
```markdown
✅ 已选择手动创建

以下是文档内容：

```markdown
[文档内容]
```

请手动保存到：./codespec/changes/{work_dir}/[文档名].md

完成后回复"已保存"，将继续下一步。
```

**选择跳过**：
```markdown
⚠️ 已跳过此步骤

⚠️ 注意：跳过此步骤可能导致文档链不完整。

是否继续下一步？
- [ ] 是，继续
- [ ] 否，返回
```

## 场景 7：步骤跳转前置条件不满足

### 触发条件

- 用户跳转到某个步骤，但前置步骤未完成
- 例如：跳转到步骤 3，但步骤 2 未完成

### 处理流程

```markdown
⚠️ 前置条件不满足

您要跳转到步骤 3（增量设计），但以下前置步骤未完成：

- [ ] 步骤 1：需求澄清（proposal.md 未生成）
- [x] 步骤 2：增量规格定义（delta-spec.md 已生成）

建议选项：
- [ ] 先完成前置步骤
  将跳转到步骤 1，完成后再进入步骤 3

- [ ] 基于现有文档继续
  ⚠️ 注意：缺少前置文档可能导致设计不完整

- [ ] 取消跳转
  返回当前步骤

请选择处理方式：
```

### 处理逻辑

**选择完成前置步骤**：
- 跳转到第一个未完成的步骤
- 完成后自动进入目标步骤

**选择基于现有文档继续**：
```markdown
⚠️ 警告：将在缺少前置文档的情况下继续

这可能影响：
- 设计的完整性
- 文档链的追溯性

确认继续吗？
- [ ] 确认继续
- [ ] 返回重新选择
```

## 场景 8：重复检测到相同工作目录

### 触发条件

- 用户开始新的设计，但检测到已有相同的工作目录
- 例如：用户说"开始设计用户注册功能"，但已存在 `US202601010015-用户注册功能`

### 处理流程

```markdown
🔍 检测到相似的工作目录

已存在以下工作目录：

- US202601010015-用户注册功能（最后修改：2026-02-11 14:30）

您是否要：

- [ ] 继续使用现有目录
  将加载现有文档，您可以继续或修订

- [ ] 创建新目录
  将创建新的工作目录（建议使用不同的编号或名称）

- [ ] 查看现有目录内容
  显示现有文档列表

请选择：
```

### 处理逻辑

**选择继续使用现有目录**：
- 加载现有文档
- 显示当前进度
- 询问用户是继续还是修订

**选择创建新目录**：
- 提示输入新的目录名
- 创建新目录
- 开始新的设计

**选择查看现有目录内容**：
```markdown
📁 现有目录内容

./codespec/changes/US202601010015-用户注册功能/
├── proposal.md ✅（已完成）
├── delta-spec.md ⏳（进行中）
├── delta-design.md ⏸️（未开始）
└── tasks.md ⏸️（未开始）

返回选择：
- [ ] 继续使用现有目录
- [ ] 创建新目录
```

## 场景 9：网络错误或 API 调用失败

### 触发条件

- 调用 CodeWiki API 失败
- 网络连接问题
- API 返回错误

### 处理流程

```markdown
❌ API 调用失败

错误信息：[具体错误信息]

建议选项：
- [ ] 重试
- [ ] 检查网络连接
- [ ] 使用离线模式（手动操作）
- [ ] 取消操作

请选择处理方式：
```

### 处理逻辑

**选择重试**：
- 重新调用 API
- 如果再次失败，提示其他选项

**选择检查网络连接**：
```markdown
📋 检查网络连接

请执行以下操作：
1. 检查网络连接：`ping codewiki.rnd.huawei.com`
2. 检查代理设置

完成后回复"已检查"，将重试。
```

**选择离线模式**：
```markdown
✅ 已切换到离线模式

请手动完成以下操作：
1. [手动操作步骤1]
2. [手动操作步骤2]

完成后回复"已完成"，将继续下一步。
```

## 场景 10：用户请求调整已完成的内容

### 触发条件

- 用户说"不对"、"我改变主意了"、"需要调整"
- 用户对已生成的内容不满意

### 处理流程

```markdown
📋 收到调整请求

您希望调整哪个部分？

- [ ] 业务价值
- [ ] 功能范围
- [ ] 用户故事
- [ ] 验收标准
- [ ] 整个文档

请选择要调整的部分：
```

### 处理逻辑

**选择某个部分**：
```markdown
✅ 已选择：[部分名称]

当前的 [部分名称]：
[当前内容]

请提供新的理解：
[用户输入]

✅ 已更新 [部分名称]

是否需要重新生成文档？
- [ ] 是，重新生成
- [ ] 否，继续
```

**选择整个文档**：
```markdown
✅ 已选择：整个文档

将重新开始 [步骤名称] 的问答和文档生成。

确认重新开始吗？
- [ ] 确认
- [ ] 取消
```

## 通用异常处理原则

### 1. 友好提示

- 使用清晰的错误信息
- 提供可行的解决方案
- 避免技术术语过多

### 2. 多选项提供

- 提供多个处理选项
- 每个选项说明利弊
- 让用户有选择权

### 3. 状态保持

- 异常处理后保持上下文
- 不丢失已收集的信息
- 能够恢复到异常前的状态

### 4. 日志记录

- 记录异常信息
- 便于后续排查
- 帮助改进 skill

### 5. 优雅降级

- 在关键功能不可用时，提供降级方案
- 确保用户能够继续工作
- 明确告知降级的影响
