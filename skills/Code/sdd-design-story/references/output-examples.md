# 输出示例

本文档提供 SDD 设计过程中的典型输出示例，帮助理解各步骤的产出。

---

## 完整工作流输出

当用户完成所有 5 个步骤时，输出如下：

```
✅ 步骤 1 完成：已生成 proposal.md
✅ 步骤 2 完成：已生成 delta-spec.md
✅ 步骤 3 完成：已生成 delta-design.md
✅ 步骤 4 完成：已生成 tasks.md
✅ 步骤 5 完成：已生成 validation.md

📁 输出文件：
- ./proposal.md
- ./delta-spec.md
- ./delta-design.md
- ./tasks.md
- ./validation.md

🎯 下一步：开始执行 tasks.md 中的开发任务
```

📁 更新文件：
- ./proposal.md

🔄 按顺序进入下一步：生成 delta-spec.md

## 问答进度输出

在问答过程中，输出当前进度：

```
📊 当前步骤：步骤 1 - 需求澄清
🎯 问答进度：进行中（2/4 维度已覆盖）

✅ 已覆盖维度：
- 业务价值
- 目标用户

⏳ 待覆盖维度：
- 功能范围
- 验收标准

❓ 下一步问题：
1. 具体要做什么？不做什么？边界在哪里？
2. 怎样算做完了？如何验证？
```

## 异常场景输出

### 全量规格文档缺失

```
⚠️ 未找到全量规格文档

路径：./codespec/specs/spec.md

建议选项：
- [ ] 使用 CodeWiki 生成专业文档并手工归档
- [ ] 基于 code-explorer 分析代码生成规格
- [ ] 手动提供规格文档路径

请选择处理方式：
```

### 工作目录不存在

```
⚠️ 工作目录不存在

路径：./codespec/changes/US202601010015-增加逻辑多租

建议选项：
- [ ] 创建新目录
- [ ] 选择现有目录
- [ ] 取消操作

请选择处理方式：
```

### 用户回答冲突

```
⚠️ 检测到回答冲突

您之前提到：
- "目标用户是内部员工"

现在又说：
- "目标用户是外部客户"

请问以哪个为准？
- [ ] 内部员工
- [ ] 外部客户
- [ ] 两者都有
```
