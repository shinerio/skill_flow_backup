## 上下文

## 你的任务

请提出 2-3 个具体的问题，帮助明确任务拆分。问题应聚焦于：

1. **任务粒度**：任务拆分是否合适？是否需要进一步细分？
2. **依赖关系**：任务之间的依赖是否清晰？有没有循环依赖？
3. **优先级**：哪些任务是关键路径？哪些可以并行？
4. **风险点**：哪些任务技术风险较高？需要预研吗？

## 约束
- 只提问，不要给出完整任务清单
- 问题要具体，针对设计增量中的功能点

## 输出格式要求

你必须以纯 JSON 格式输出澄清问题（不要包含任何其他文字），格式如下：

{
  "title": "任务分析",
  "questions": [
    {
      "id": "q1",
      "type": "single_choice",
      "label": "任务拆分的粒度偏好？",
      "options": ["细粒度（每个任务 1-2 小时）", "中粒度（每个任务半天）", "粗粒度（每个任务 1-2 天）"],
      "required": true
    },
    {
      "id": "q2",
      "type": "multiple_choice",
      "label": "哪些模块需要优先完成？",
      "options": ["核心业务逻辑", "用户界面", "数据存储", "测试用例"],
      "required": true
    }
  ]
}

支持的问题类型：
- text: 单行文本输入
- textarea: 多行文本输入
- single_choice: 单选（必须提供 options 数组）
- multiple_choice: 多选（必须提供 options 数组）

注意：直接输出 JSON，不要用 markdown 代码块包裹。
