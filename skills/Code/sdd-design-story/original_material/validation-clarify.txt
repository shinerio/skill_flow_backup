## 当前阶段：一致性验证（第 5/5 步）
## 当前任务：一致性检查

## 上下文
- 你**必须**参考 SPEC.md 和 DESIGN.md 了解基线文档
- 你**必须**参考当前 AR 的所有文档：proposal.md、delta-spec.md、delta-design.md、tasks.md

## 你的任务

请进行初步的一致性检查，指出发现的问题。检查维度：

1. **Proposal ↔ Delta-Spec**：提案中的功能点是否都有对应规格？
2. **Delta-Spec ↔ Delta-Design**：规格是否都有对应设计？
3. **Delta-Design ↔ Tasks**：设计是否都有对应任务？
4. **异常场景**：是否有遗漏的异常处理？
5. **边界条件**：是否有遗漏的边界情况？

## 约束
- 指出具体问题，不要泛泛而谈
- 给出问题的严重程度（高/中/低）
- 如果没有发现问题，也要说明检查了哪些方面

## 输出格式要求

你必须以纯 JSON 格式输出澄清问题（不要包含任何其他文字），格式如下：

{
  "title": "一致性验证",
  "questions": [
    {
      "id": "q1",
      "type": "single_choice",
      "label": "发现 delta-spec 中缺少异常处理规格，是否需要补充？",
      "description": "严重程度：高",
      "options": ["是，现在补充", "否，后续迭代处理", "不需要，已有其他机制处理"],
      "required": true
    },
    {
      "id": "q2",
      "type": "textarea",
      "label": "请补充遗漏的边界条件说明",
      "description": "严重程度：中",
      "required": false
    }
  ]
}

支持的问题类型：
- text: 单行文本输入
- textarea: 多行文本输入
- single_choice: 单选（必须提供 options 数组）
- multiple_choice: 多选（必须提供 options 数组）

注意：直接输出 JSON，不要用 markdown 代码块包裹。
